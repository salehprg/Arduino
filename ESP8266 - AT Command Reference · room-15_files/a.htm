                                                  I can confirm what others has said regarding +IPD to recieve data.
                                                  As an example:
                                                  You open up a tcp connection to your server and use CIPSEND to send "PING\r\n". On the server end you respond with PONG\r\n.
                                                  This will give the following output from ESP8266:
                                                  SEND OK

                                                  +IPD,6:PONG

                                                  OK

                                                  Where 6 is the number of bytes in response followed by a : and the data.
                                                  Works very well.

                                                        • lintilla10 Abe Cross a year ago

                                                          Well this is not a traditional ICMP ping if thats what you think. You need to make a socket server listening for the data on the remote end.

                                                          But after connecting to the wifi with AT+CWJAP="myssid", "mypass":

                                                          AT+CIPSTART="TCP", "myip", myport\r\n

                                                          AT+CIPSEND=6\r\n

                                                          > PING\r\n

                                                          Your socket server in the other end then needs to respond with PONG\r\n

                                                          The result will be as above, with a +IPD,6:PONG from the ESP8266

                                                          If you want to do a ICMP ping you need to use some kind of library since the ESP does not include this function.

                                                          E.g. https://github.com/dancol90... (I have not tested this myself).
